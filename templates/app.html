Account

Notification settings

Choose how Sinau pings you about task deadlines. Notifications are off by default — turn them on to start receiving reminders.

Reminders go out at most once per task per day. Mentors get notifications for tasks they have assigned; learners get them for tasks assigned to them. If the chosen channel is not yet wired by the operator (no SMTP server, no WhatsApp gateway, no Telegram bot), reminders quietly fall back to the server log instead of failing. WhatsApp and Telegram are listed as preview: the routing is in place but the actual delivery integration is not yet enabled.

Getting started

How Sinau works

Sinau is a private learning tracker with two room formats: Mentorship for progress check-ins and Classroom for assignments, submissions, and review.

{{if .SetupNeeded}}Create first room{{end}} {{if not .User}}LoginJoin with invite{{end}}

Role guide

If you are a mentor

  1. Set up the first mentor account and your first room (only the first user can do this).
  2. Choose a room format. Use Mentorship for guided cohorts with check-ins and tasks. Use Classroom for teacher/student assignment submission workflows.
  3. Invite learners from the room page — each code is single-use and expires after seven days.
  4. Review reports. Learners post weekly check-ins. Comment, flag blockers, keep them moving.
  5. Assign tasks with optional deadlines. Pick a single learner or "All learners" to fan one task out to every learner in the room.
  6. Review completed tasks on a fixed 1–5 scale (1 = needs more work, 3 = good, 5 = exceptional). Same scale across every task → leaderboards stay comparable.
  7. Toggle the leaderboard for each cohort in Room settings — off keeps it private, on shows the full ranking to learners.

Room formats

Mentorship vs Classroom

  1. Mentorship gives learners a progress form, blockers, resource links, mentor comments, deadline tasks, points, and optional leaderboards.
  2. Classroom gives mentors assignment publishing, deadline fields, student submission links, review status, feedback, scores, invites, and a class roster.
  3. Separate rooms keep workflows clean. A mentor can create multiple rooms, so one cohort can be mentorship-style while another is classroom-style.

Role guide

If you are a learner

  1. Join with the invite code your mentor sent you.
  2. Submit progress regularly — what you learned, practiced, your blocker (if any), and the next concrete step.
  3. Work through tasks: move them todo → doing → done. Once done, your mentor reviews and awards points.
  4. Track your score on the room page. If the mentor has enabled it for the cohort, you also see the leaderboard.
{{if .NotificationsEnabled}}

Stay in the loop

Notifications

Reminders are off by default. Open Settings after logging in to enable deadline reminders and pick a channel (email is the standard option). Both mentors and learners can opt in — only the ones who turn it on get pinged. Reminders fire at most once per task per day.

{{end}}

Why points

Gamification, fairly

Every task is reviewed on the same 1–5 rubric, so cohort leaderboards reflect output rather than how generous a particular mentor is feeling. Mentors set leaderboard visibility per room.
